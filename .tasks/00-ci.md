# 00 — CI

Depends on: nothing
Unblocks: everything (green/red signal for all later phases)

## Why

The repo has a GitHub upstream but no CI. Pre-commit covers fmt/clippy
locally; nothing runs tests on push. The toolchain is a pinned nightly
(`rust-toolchain.toml`: `nightly-2026-05-28`), so breakage from a toolchain
bump or a nightly-feature change should be caught by a bot, not discovered
mid-session.

## Requirements

- GitHub Actions workflow on push and PR to `main`.
- Steps (mirror the pre-commit hooks plus tests):
  - `cargo fmt --all -- --check`
  - `cargo clippy --all --all-targets --all-features -- -D warnings`
  - `cargo test --all`
- The toolchain must come from `rust-toolchain.toml` (rustup respects it
  automatically; if a setup action is used, make sure the pinned file wins).
  Components needed: `rustfmt`, `clippy`.
- Cache the cargo registry and build artifacts, keyed on `Cargo.lock` and the
  toolchain pin.
- `.cargo/config.toml` uses `-Z threads=8` and the workspace uses
  `cargo-features = ["profile-rustflags"]` — both nightly-only and fine on
  the pinned toolchain; do not "fix" them for stable.

## Open questions

- Single job vs. split lint/test jobs — either is fine; optimize wall time.
- Whether to add a Nix job (`nix flake check` or building the dev shell) —
  ask the user whether Nix CI is worth the maintenance.

## Out of scope

- Running benches in CI (`--all-targets` already compile-checks them).
- Release/publish automation.

## Done when

A trivial commit pushed to a branch shows a green workflow run on GitHub.
The user pushes; signal when the workflow file is ready.
